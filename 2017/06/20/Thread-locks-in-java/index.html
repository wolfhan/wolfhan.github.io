Thread locks in java

Posted on 2017-06-20 | Edited on 2019-01-08 | In java |

In java, synchronzation is a really important thing. To use it, you must be familiar with the intrinsic lock or monitor

at first.

What is it

The intrinsic lock is an internal object ensuring that only one thread access the shared resource at a time. Once a

thread gets the lock, the other therads can not accquire it at once until the synchronized method or statement releases

it.

it will get different results that are caused by thread interference or data consistency. So, we need to change it so

that it does not result in such confusion. And synchronzied method or synchronzied statement will do that.

How to use it

At first, we can use the synchronzied method to change the code above.

And now, it creates the same result whatever the call order of that three threads is. Why can the synchronzied methods

solve these problems? In fact, these synchronzied methods in these threads accquire the lock of the object at first. And

only one thread can accquire exclusively the lock object at a time. Other synchronzied methods could accquire the

intrinsic monitor only if after the method occupying the lock releases it.

Synchronzied statements use the same thing to do that. We can use the synchronzied statements to change the original class.

Synchronzied statements also accquire the lock of the object at first, and then only the method accquiring that lock can

be executed immediately.

Other lock

Although intrinsic locks can solve thread interference and data constitency, it also create other problems, such as dead

lock caused by that the two threads want to use the same lock but cannot occupy it neither.

Since jdk version is greater than 1.5, java brings the new lock that not resulting in dead lock. Every time we can use

tryLock() to accquire a lock, and use unlock() to release it after using up. We can use the new lock to change the

original code.

In conclusion, you can use intrinsic locks that is used by synchronzied methods or statements or new concurrent lock

in the multithread programming.
