java nio introduction

Posted on 2017-06-12 | Edited on 2019-01-09 | In java |

We usually use classes in java.io package to manage io operations, but java.nio can be much powerful

than it.

What is nio

java.nio is a package related to io operations. There are two main class in it: Path and Files.

Path is a class that is about paths, such as resolving a path, relativizing a path and comparing two

paths. Files is about file management, such as file creation, file deletion, file modification

and file metadata.

Why we need nio

In java, we can use java.io.File class to execute all file-related operations. So, why will we need

nio? The reason is simple. File class in io package has various kinds of file operations, but it is not enough

and is limited, for example, its rename method is not platform independent, it can not manage file

meta data, such as file permission, file owner, and so on. That’s why that we need a new class to

do so, and nio package plays the role. Nio package is a brand new class that refactors the java.io

package. In other words, you can use nio package to do anything that java.io can do, and do something

that java.io cannot do.

How to use nio

It’s easy to use nio package. To use Path class, you can create a path through Path path = Paths.get("some_path").

Then, you can resolve a path, relativize a path, or do other things based on that. For instance, we

need resolve a path based a parent path, so we can use

Path parent = Paths.get("parent_path"); Path path = parent.resolve("some_path");.

And also we can get absolute path, get real path or compare two paths through Path class.

We can use Files class to manage file operations, such as file creation, file deletion, file modificaiton

and file attributes. We can also use it determine whether a file is a directory, a regular file or symbolic link

or not. For example, if we need to determine whether a file is a regular file, we can use Files.isRegularFile(path);.

If we need to delete a file, we can do this by Files.delete(path);.

Above all, nio package is a very powerful package that can do many things for you. And if you would like use

nio package with your legacy io code, you can use the conversion File.toPath();.
